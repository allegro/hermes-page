Fast and reliable message broker built on top of Kafka

Sound foundations

Hermes builds on widely adopted OpenSource tools - Kafka and Zookeeper.

Reliable

We have been using Hermes in production for mission-critical messaging for 6+ months, processing billions of events and solving many corner-cases. We are constantly monitored by our users looking for 100% delivered events.

Stable response time

Ever needed strict SLA for published events? Hermes takes care of any Kafka hiccups and can guarantee maximum response time for 99th percentile of published events.

Fault tolerant

Network partitioning? Kafka having trouble? Hermes can endure it, staying responsive to clients and sending events to Kafka when problems are over.

Low overhead

Hermes is just a thin wrapper over Kafka - we manage to make lives of developers easier at low cost.

Easy to use

It takes HTTP client to become a publisher and HTTP endpoint to become a subscriber - it can't get any easier then that.

We care about subscribers

Changing pull to push model, we created algorithms that adapt sending speed to subscribers current capabilities.

No longer a black box

We gather a lot of metrics that can be sent to Graphite - track each topic and subscription rate, latency, Kafka lag. We also provide tools that allow for tracking state of each event on chosen topic/subscriptions when closer look is needed.

Easy to manage

We created GUI to manage Hermes (and Kafka) topics and subscriptions. API simplifies common operations like repeating already received events - just give us a timestamp and we will find relevant events on all partitions.

Easily extensible

Hermes components can be used as libraries, with programmatic API that can be easily extended and customized. You can register own callbacks, apply authorization and many more.

Easy to get started with

Thanks to Docker magic, you are able to run Hermes on your localhost in minutes - check out our getting started guide for more.
